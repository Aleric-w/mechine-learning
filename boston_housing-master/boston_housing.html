编程练习3：定义衡量标准Â¶

如果不能对模型的训练和测试的表现进行量化地评估，我们就很难衡量模型的好坏。通常我们会定义一些衡量标准，这些标准可以通过对某些误差或者拟合程度的计算来得到。在这个项目中，你将通过运算决定系数 R2 来量化模型的表现。模型的决定系数是回归分析中十分常用的统计信息，经常被当作衡量模型预测能力好坏的标准。

R2的数值范围从0至1，表示目标变量的预测值和实际值之间的相关程度平方的百分比。一个模型的R2 值为0还不如直接用平均值来预测效果好；而一个R2 值为1的模型则可以对目标变量进行完美的预测。从0至1之间的数值，则表示该模型中目标变量中有百分之多少能够用特征来解释。模型也可能出现负值的R2，这种情况下模型所做预测有时会比直接计算目标变量的平均值差很多。

在下方代码的 performance_metric 函数中，你要实现：

  • 使用 sklearn.metrics 中的 r2_score 来计算 y_true 和 y_predict的R2值，作为对其表现的评判。
  • 将他们的表现评分储存到score变量中。

或

  • (可选) 不使用任何外部库，参考决定系数的定义进行计算，这也可以帮助你更好的理解决定系数在什么情况下等于0或等于1。

def performance_metric(y_true, y_predict):
    """计算并返回预测值相比于预测值的分数"""
    from sklearn.metrics import r2_score
    score = r2_score(y_true,y_predict)

    return score

def performance_metric2(y_true, y_predict):
    """计算并返回预测值相比于预测值的分数"""
    a =0
    ssr,sst =0,0
    mean_y = np.average(y_true)
    if (len(y_true) == len(y_predict)):
        while a < len(y_true):
            ssr,sst = ssr + np.square(y_true[a] - y_predict[a]),sst + np.square(y_true[a] - mean_y)
            a = a+1
    else:
        raise Exception('length not matched(y_true)(y_predict)')    
    score = 1 - ssr/sst
    return score

问题 7- 网格搜索（Grid Search）Â¶

什么是网格搜索法？如何用它来优化模型？

问题 7 - 回答:Â¶

网格搜索法是指，尝试各种指定的参数对值来进行交叉验证，找出交叉验证精确度最高的参数对值作为模型最优参数的参数搜索方法。

GridSearchCV中 param_grid 传入不同参数枚举的参数数据，param_grid对于不同的分类器有不同的参数，例如 svm.SVC()中包括 kernel_parameter , C , gamma 等参数，tree.DecisionTreeRegressor分类器有max_depth，min_samples_leaf，max_leaf_nodes等参数 。然后使用param_grid中的参数对可能值，计算 scoring 中的评判方法，得出最优参数，输出其中的最优参数对。

网格搜索法一般用于不确定模型最优参数的情况下，通过枚举可以优化模型的参数对值，交叉验证各种最后得出其中的最优参数。

问题 8 - 回答：Â¶

K折交叉验证法是先将训练集分成K份，轮流选取其中1个子集作为验证集，其余的 k-1个子集作为训练集的验证方法。K折交叉验证法，可以保证每一个数据都可以经过训练，并且会作为验证集验证一次。

网格搜索法是选择模型最优参数的方法，如果只是用选择部分数据作为验证集，在训练和验证过程中，计算最优参数时会受到数据分布的影响，并不能最优化参数的选择。当结合K折交叉验证，所有数据都会被用于训练和验证，可以消除在训练和验证过程中数据分布对最优参数的影响。

不同参数的组合方式及其计算结果以字典的形式保存在 'cvresults'中。可以使用print输出里面的结构，数据类型是pandas DataFrame结构，包括'mean_fit_time', 'mean_score_time', 'mean_test_score','mean_train_score', 'param_C', 'param_kernel', 'params','rank_test_score', 'split0_test_score','split0_train_score', 'split1_test_score', 'split1_train_score', 'std_fit_time', 'std_score_time', 'std_test_score', 'std_train_score'...等要素，主要是记录每一次交叉验证的对应属性值。

网格搜索如果不使用交叉验证，模型准确率会受到数据偏斜分布和训练集选取分布的影响。交叉验证可以确保每一个数据都能经过训练和验证，不会受到数据偏斜分布的影响。
